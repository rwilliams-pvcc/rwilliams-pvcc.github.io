Networking & The Internet

TERM DEFINITION
Network Classifications
  • LAN
  • MAN
  • WAN
 Classification of networks based on whether the network's internal operation is based on designs that are in the public domain or on innovations owned and controlled by a particular entity such as an individual or corporation.
  • Local Area Network.
  • Metropolitan Area Network.
  • Wide Area Network.
Network Models
  • bus network
  • star network
 Another way of classifying networks based on the topology of the network.
  • All machines are connected to a common communication line called a bus.
  • One machine serves as a central focal point to which all others are connected.
Protocols
  • CSMA/CD Ethernet
  • CSMA/CA Wifi
 The established rules by which activities are conducted for a network to function reliably.
  • Stands for Carrier Sense, Multiple Access with Collision Detecion. This protocol dictates that each message be broadcast to all machines on the bus.
  • Stands for Carrier Sense, Multiple Access with Collision Avoidance. Wireless networks adopt the policy of trying to avoid collisions rather than trying to detect them.
Combining Networks
  • repeater
  • bridge
  • switch
  • router
  • gateway
 Sometimes it is necessary to connect existing networks to form an extended communication system. This can be done by connecting the networks to form a larger version of the same "type" of network.
  • A little more than a device that passes signals backa nd forth between the two original buses (usually with some form of amplification) without considering the meaning of the signals.
  • It looks at the destination address that accompanies each message and forwards a message across the connection only when that message is destined for a computer on the other side.
  • Essentially a bridge with multiple connections, allowing it to connect to several buses rather than just two.
  • The connection between networks to form an internet is handled by these devices. Routers are special purpose computers used for forwarding messages.
  • The "point" at which one network is linked to an internet. Serves as a passageway betwen the network and the outside world.
Client/server model
  • P2P: peer-to-peer
  • distributed system
  • cloud computing
 This model defines the basic roles played by the processes as either a client, which makes requests of other processes, or a server, which satisfies the requests made by clients.
  • P2P model involves processes that provide service to and receive from each other. Usually involves processes that execute on a temporary basis.
  • Consist of software units that execute processes on different computers.
  • Huge pools of shared computers on the network can be allocated for use by clients as needed. The Internet is making it possible for entities to entrust their data and computations to "the Cloud."
Internet Addressing
  • Tier 1
  • Tier 2
  • Tier 3
 An internet needs an internet-wide addressing system that assigns a unique identifying address to each computer in the system.
  • Tier-1 ISPs (Internet Service Providers) consist of very high-speed, high-capaticity, international WANs. Thought to be the backbone of the Internet. Typically operated by large companies that are in the communications business.
  • Tier-2 ISPs tend to be more regional in scope and less potent in their capabilities. These are also typically operated by companies in the communications business.
  • Tier-3 ISP is essentially an independed Internet operated by a single authority that is in the business of supplying Internet access to individual homes and businesses.
IP Address
  • ICANN
  • DNS
  • TLD: Top-level domain
  • Country-code TLDs
 Is the unique identifying address to each computer in the system. Originally was a pattern of 32 bits, but in the process of converting to 128-bit addresses.
  • Blocks of consecutively numbered IP addresses are awarded to ISPs by the Internet Corporation for Assigned Names and Numbers (ICANN), which is a nonprofit corporation established to coordinate the Internet's operation.
  • Stands for Domain Name System and is an Internet-wide directory system of name servers.
  • The suffix that follows the period in a domain and is used to reflect the domain's classification. For example, .edu means educational.
  • The Internet quickly grew to include countries all over the world. Country-code TLDs refer to the specific country. For example, .au for Australia.
Internet Protocols
  • FTP
  • HTTP
  • SMTP
 Set of rules that the Internet follows to transfer data.
  • File Transfer Protocol, an application for listing and copying files across the network.
  • Hyper Text Transfer Protocol. Traditional network applications have come to be handled by webpages via HTTP.
  • Simple Mail Transfer Protocol defines a way that two computers on the network may interact when transmitting an email message from one host to another.
World Wide Web
  • URL
  • client-side activities
  • server-side activities
 Started as a hypertext document format for embedding hyperlinks to other documents; a protocol for transferring hypertext across the network, and a server process that supplied hypertext pages upon request.
  • In order to locate and retrieve documents on the Web, each document is given a unique address called a Uniform Resource Locator (URL). Each URL contains the information needed by a browser to contact the proper server and request the desired document.
  • Performed by a client. What the client-side sees, like images or an order form.
  • Performed by a server. What the server-side sees, responsible for the coding that links the image or getting the details from the order form.
Internet four-layer software
  • application layer
  • transport layer
  • network layer
  • link layer
  • A message typically originates here.
  • Accepts messages from application layer and ensure that the messages are properly formatted for transmission over the Internet.
  • Decides in which direction a packet should be sent at each step along the packet's path through the Internet.
  • A message is received by the link layer at the destination and passed back up the hierarchy until it is delivered to the application layer at the message's destination.
Internet is:
  • packet-switching network
  • fault tolerant
  • redundant
  • Digital data is sent by breaking the data into blocks of bits called packets, which contain both the data being transmitted and control information for routing the data.
  • Allows the system to continue operating when there are failures.
  • Process of multiple paths for traffic so that data can continue to move in the case of a failure.
Cybersecurity
  • malware
  • virus
  • worm
  • Trojan horse
  • spyware
  • phishing
  • DDoS attack
 The study of protecting computers, networks, and data from attacks.
  • Malicious software that is used to attack systems.
  • A software that infects a computer by inserting itself into programs that already reside in the machine. Then, when the "host" program is executed, the virus is also exectued.
  • An autonomous program that transfers itself through a network, taking up residence in computers and forwarding copies of itself to other computers.
  • A program that enters a computer system disguised as a desirable program, such as a game or useful utility package, that is willingly imported by the victim.
  • A software that collects information about activities at the computer on which is resides and reports that information back to the instigator of the attack.
  • A technique of obtaining information explicity by simply asking for it. Often carried out via email.
  • A computer in the network can be attacked by software being executed on other computers in the system. Stands for Denial of Service and is the process of overloading a computer with messages. Generally used against large commercial webservers to disrupt the company's business.
Protection
  • firewall
  • spam filter
  • proxy server
  • antivirus software
  • cryptography
  • public key encryption
 Different types of protection available to help prevent cyber attacks, like the ones mentioned above.
  • A primary prevention technique to filter traffic passing through a point in the network.
  • Firewalls that are used to block unwanted emails.
  • A software unit that acts as an intermediary between a client and a server with the goal of shielding the client from adverse actions of the server.
  • Used to detect and remove the presence of known viruses and other infections.
  • The study of sending and receiving messages securely in the presence of adversaries.
  • Involves techniques by which the encryption and decrpytion algorithms are known, and one of the keys is known, but the encoded message remains secure.
